The basics

WebPicture lets you create awesome DSL web based editors, using key features of EMF and Picture a domain specific language independent to the desire metamodel to represent.

Getting everything you need

Building the metamodel

EMF technology lets you create quick and easy metamodels for code generation. WebPicture requieres an ecore model you can learn more about EMF you can follow this tutorial.

Setting the graphical representation

Currently WebPicture supports two ways to set graphical representations for models.

One way to represent your models is to choose the default set option, by choosing this option WebPicture takes a basic default set to get everything painted. You can learn more about this option in our default graphical set site.

The second option to set the graphical representation for your model is to upload your own graphical representation. You can write down your own Picture file following the specification available in the EnAr Wiki . Also there is a cool option that you must try it auto generates a custom Picture file based on a ecore file you can choose between several options to customize everything in your editor. You can learn more at PictureMaker.

If your Picture files references media resources such as images you must upload a folder which contains all the resources requiered. We recomend using a single folder that contains all your files.

Generating the editor

Finally to wrap things up give a name to your new editor, get credit for it and give it a description.

Ready

Got everything done now click the Generate editor button to create the new editor.
